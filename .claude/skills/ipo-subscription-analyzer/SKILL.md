# IPO 打新分析师

description: 港美股 IPO 打新综合分析。扫描当天/近期可申购 IPO 标的，进行多维度分析（基本面、Red Flags、板块热度、基石投资者、保荐人、A/H 关联、暗盘情绪），评估中签率与资金分配，输出综合打新建议。Triggers on "IPO 分析", "打新", "今日IPO", "今天打新", "IPO subscription", "新股申购", "IPO calendar", or "upcoming IPO". 当用户说"今日IPO分析"等不指定市场时，默认扫描港股+美股+A股全部市场并对比。当用户用中文询问时触发。支持港股/美股/A股 IPO 市场分析。

## 数据源策略

IPO 数据高度时效性，采用 **API 结构化数据 + WebSearch 补全** 双轨模式：

### ⚠️ 港股 IPO 数据时效性警告

港股 IPO 招股信息存在严重的公开搜索延迟问题：
- 券商 APP（富途/老虎/盈透）通常在招股当天即更新，但其数据不可被 WebSearch 抓取
- 港交所 HKEXnews PDF 公告虽然实时，但 WebSearch 对 PDF 内容的索引有 1-3 天延迟
- 财经媒体（东方财富/财联社/新浪）报道通常延迟 0.5-2 天
- **结果**：刚开始招股的新股可能在 WebSearch 中搜不到招股细节（入场费/每手/发售价）

**应对策略：**
1. WebSearch 搜索时使用多个时间窗口查询（"今日"、"本周"、"最新"）
2. 直接搜索 HKEXnews 公告 PDF（`site:hkexnews.hk [公司名] 招股章程`）
3. 搜索券商平台新股页面（`"[公司名]" 打新 入场费 孖展`）
4. 若搜索无果，在报告中明确标注"数据待更新，建议查阅券商 APP 确认"

### 各市场数据源覆盖

| 市场 | 结构化 API | WebSearch | 说明 |
|------|-----------|-----------|------|
| **美股** | Finnhub `ipo_calendar` / FMP `ipo_calendar` | Nasdaq / IPOScoop | API 优先，WebSearch fallback |
| **A 股** | AKShare `stock_new_ipo_cninfo()` / `stock_zh_a_new_em()` | 东财 / 同花顺 | API 优先，WebSearch fallback |
| **港股** | **无免费 API** | 富途 / 东财港股 / AAStocks / HKEX | **WebSearch 为唯一数据源，存在延迟** |

> 港股 IPO 缺乏免费结构化 API。Finnhub/FMP 仅覆盖美股，AKShare 仅覆盖 A 股。
> 未来可考虑集成 [Futu OpenAPI](https://www.futunn.com/OpenAPI) `GetIPOList(market)` 实现港股结构化数据（需运行 FutuOpenD 网关）。

| 数据源 | 市场 | 端点/函数 | 用途 |
|--------|------|----------|------|

| 市场 | 搜索查询模板 | 优先数据源 |
|------|-------------|-----------|

## Workflow

### Step 1: IPO 日历扫描

**市场范围判定规则（严格执行）：**

| 用户指令示例 | 扫描范围 | 说明 |
|-------------|---------|------|
| "港股打新"、"港股IPO"、"今日港股IPO分析" | **仅港股** | 含"港股"关键词 → 只扫描港股 |
| "美股IPO"、"US IPO"、"美股打新" | **仅美股** | 含"美股/US"关键词 → 只扫描美股 |
| "A股新股"、"A股打新"、"今日A股申购" | **仅A股** | 含"A股"关键词 → 只扫描 A 股 |
| "今日IPO分析"、"打新分析"、"IPO分析" | **全市场** | 无市场限定词 → 扫描港股+美股+A股 |

> **重要：** 当用户指定了具体市场时，**禁止**扫描其他市场。例如用户说"港股打新"，则只输出港股 IPO 标的，不要附加美股或 A 股信息。跨市场对比（Step 9 第 0 项）仅在全市场扫描时输出。

**1a. 美股 — API 优先（仅当扫描范围含美股时执行）：**
- 调用 Finnhub `ipo_calendar` 或 FMP `ipo_calendar` 获取未来 2 周 IPO 列表
  - 参数: `from={today}`, `to={today+14}`
  - 过滤: `status` = "expected" 或 "priced"
- API 失败时 fallback: WebSearch `"US IPO calendar this week {year}"`

**1b. A 股 — API 优先（仅当扫描范围含 A 股时执行）：**
- 调用 AKShare `stock_new_ipo_cninfo()` 获取近期新股
- API 失败时 fallback: WebSearch `"A股新股申购 本周 {年月}"`

**1c. 港股 — WebSearch（仅当扫描范围含港股时执行）：**
- `"港股IPO 本周招股 {当前年月}"` / `"Hong Kong IPO calendar this week {year}"`
- 优先从富途、东方财富、AAStocks 提取结构化数据

**提取字段（每只标的）：**
- 公司名称（中英文）、股票代码
- 招股价区间、每手股数、入场费
- 申购起止日期、定价日、暗盘日（港股）、上市日
- 保荐人/承销商、募资规模
- 当前状态（招股中/已截止/已定价/已上市）

输出为日历总览表：

| 代码 | 公司名称 | 市场 | 招股价 | 入场费 | 申购截止 | 上市日 | 募资规模 | 状态 |
|------|---------|------|--------|--------|---------|--------|---------|------|

### Step 2: 标的基本面分析

**搜索查询：**
- `"[公司名] 招股书 收入 利润"` / `"[公司名] IPO prospectus revenue profit"`
- `"[公司名] IPO 估值 市盈率"` / `"[公司名] IPO valuation PE ratio"`
- `"[公司名] S-1 filing"` （美股，SEC EDGAR）

**分析维度：**
- 收入规模与增长率（近 3 年，>30% = 强劲，<10% = 疲弱）
- 净利润/亏损与趋势（路径至盈利可见性）
- 毛利率与同行业对比（>60% = 优秀，<30% = 偏弱）
- 市盈率/市销率 vs 已上市可比公司（折价/溢价判断）
- 业务模式与竞争壁垒（TAM、市场地位、护城河）
- 客户集中度风险（前 5 大客户收入占比）
- 现金流状况与烧钱速率（现金跑道 >24 月 = 健康）

### Step 3: 招股书 Red Flags 检测

**搜索查询：**
- `"[公司名] 招股书 风险因素"` / `"[公司名] IPO risk factors"`
- `"[公司名] IPO 关联交易 内部人"` / `"[公司名] related party transactions"`

**Red Flags 检查清单：**

| 级别 | Red Flag | 说明 |
|------|----------|------|
| 严重 | 持续经营疑虑 | Going concern 警告 |
| 严重 | 内控重大缺陷 | Material weakness in internal controls |
| 严重 | 监管调查/诉讼 | 正在进行的重大诉讼或监管调查 |
| 高 | 关联交易占比高 | Related party transactions > 10% 收入 |
| 高 | 内部人大量抛售 | 创始人/管理层借 IPO 套现占比 > 30% |
| 高 | 募资用途模糊 | "一般公司用途" 占比过高 |
| 中 | 双重股权结构 | 创始人超级投票权（评估合理性） |
| 中 | 关键客户流失 | 近期失去重要客户 |
| 低 | 标准模板风险 | 行业通用风险披露（可忽略） |

**募资用途评估：**
- 研发/产能扩张 = 正面信号
- 偿还债务 = 中性偏负面
- "一般公司用途" = 负面信号
- 内部人抛售占比 = 重点关注

**治理评估（美股适用）：**
- 董事会独立性
- 股权结构（AB 股、反收购条款）
- 管理层锁定期

### Step 4: 板块与长期价值

**搜索查询：**
- `"[所属行业] 板块 前景 {年份}"` / `"[industry] sector outlook {year}"`
- `"[公司名] 竞争对手 市场份额"`

**分析内容：**
- 所属板块近期市场热度（涨跌幅、资金流向）
- 行业生命周期阶段（萌芽/成长/成熟/衰退）
- 可比上市公司估值区间（P/E、P/S、EV/EBITDA）
- 政策环境（监管利好/利空）
- 板块热度评分：高热 / 中等 / 冷门

### Step 5: A/H 股关联分析（港股适用）

**搜索查询：**
- `"[公司名] A股 上市 代码"` / `"[公司名] AH 溢价"`

**分析内容：**
- 是否已有 A 股上市（若有，给出 A 股代码与当前价格）
- AH 溢价/折价率参考（恒生 AH 溢价指数基准）
- 两地定价差异对 IPO 定价的锚定效应
- 若无 A 股对标，寻找最接近的 A 股可比公司

### Step 6: 基石投资者与保荐人评估

**搜索查询：**
- `"[公司名] 基石投资者 cornerstone"` / `"[公司名] IPO cornerstone investors"`
- `"[保荐人] IPO 历史表现 首日涨幅"` / `"[sponsor] IPO track record"`
- `"[公司名] 暗盘 预期"` / `"[公司名] grey market premium"`

**基石投资者评分 (0-10)：**

| 类型 | 分值 | 示例 |
|------|------|------|
| 顶级主权基金/长线基金 | 8-10 | GIC、淡马锡、Capital Group、Fidelity |
| 知名产业资本/战略投资者 | 6-8 | 行业龙头、上下游巨头 |
| 中型机构/家族办公室 | 4-6 | 区域基金、知名家办 |
| 关联方/短期对冲基金 | 1-3 | 控股股东关联公司、短线基金 |
| 无基石投资者 | 0 | — |

**加分项：** 锁定期 > 12 个月 (+1)、基石认购占比 > 50% (+1)

**保荐人评估：**
- 近 2 年保荐 IPO 数量与首日表现
- 破发率统计
- 平均超额认购倍数

**暗盘/GMP 情绪指标（港股）：**
- 暗盘溢价 > 10% = 市场看好
- 暗盘平价或折价 = 谨慎信号
- 搜索 `"[公司名] 暗盘价 grey market"` 获取实时暗盘报价
- **时效性说明：** 暗盘交易通常在上市前一个交易日 16:15-18:30 进行（如 3 月 20 日上市则 3 月 19 日暗盘）。招股期间暗盘数据尚不可用，此时应标注"暗盘待出"并基于认购热度、基石阵容等间接指标预判情绪

### Step 7: 打新机制分析

#### 港股打新

**搜索查询：**
- `"[公司名] 认购倍数 超额认购"` / `"[公司名] subscription rate"`
- `"[公司名] 中签率 一手"` / `"[公司名] allotment result"`

**分析内容：**
- 甲组（≤500 万港元）/ 乙组（>500 万港元）分组机制
- 1 手中签率预估（参考同类 IPO 历史数据）
- 多手申购中签率递增分析
- 回拨机制触发条件（15x/50x/100x 超额认购）
- 融资申购：融资倍数（通常最高 10 倍）、利息成本（年化）、资金占用天数
- 入场费 = 每手股数 × 招股价上限 × (1 + 手续费 1%)
- 预估每手盈亏 = (暗盘预期价 - 招股价) × 每手股数 - 手续费

#### 美股 IPO

**分析内容：**
- 承销商阵容评级（Goldman Sachs/Morgan Stanley/JPMorgan = 顶级）
- 定价区间 vs 预期最终定价方向（上调/维持/下调）
- 散户可参与渠道（富途/老虎/盈透 IPO 申购）
- 首日涨幅预期（参考同行业近期 IPO）
- 锁定期到期风险（通常 90-180 天）
- 内部人抛售占比（IPO 股份中老股 vs 新股比例）

#### A 股打新

**分析内容：**
- 网上申购上限（股数）
- 发行市盈率 vs 行业平均市盈率（>23 倍需关注）
- 中签缴款日、上市日
- 预估开板涨幅（参考同期新股表现）

### Step 8: 资金冲突与时间表

整合 Step 1 数据，分析多只 IPO 之间的资金冲突：

**时间线甘特图（文本版）：**
```
标的A: |---申购---|定价|暗盘|上市|
标的B:     |---申购---|定价|暗盘|上市|
标的C:         |---申购---|定价|上市|
冲突:     |===重叠期===|
```

**分析内容：**
- 申购期重叠标记（资金无法复用）
- 资金占用天数计算（申购日 → 退款日/上市日）
- 资金效率排序（占用天数短 + 收益预期高 = 高效率）
- 建议申购优先级排序

### Step 9: 综合打新建议

#### 打新评分卡（每只标的 0-100 分）

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 基本面质量 | 20% | 盈利能力、增长性、现金流 |
| 估值合理性 | 15% | vs 可比公司折价/溢价 |
| 基石/保荐人质量 | 15% | 基石评分 + 保荐人历史 |
| 板块热度 | 10% | 板块涨跌幅、资金流向、政策面 |
| 认购热度/GMP 情绪 | 10% | 超额认购倍数、暗盘溢价、公开认购占比 |
| Red Flags 扣分 | 10% | 严重 Red Flag 直接降至 <40 |
| 市场时机 | 10% | 大盘环境、近期 IPO 表现 |
| A/H 溢价机会 | 5% | AH 定价差异带来的套利空间 |
| 募资用途与治理 | 5% | 用途合理性 + 治理结构 |

#### 输出格式

**0. 跨市场对比总览（仅全市场扫描时输出，单市场扫描时跳过此项）：**

当且仅当用户未指定市场、触发全市场扫描时，输出跨市场对比表，按总分降序排列：

| 排名 | 代码 | 公司 | 市场 | 板块 | 总分 | 建议 | 入场费 | 申购截止 |
|------|------|------|------|------|------|------|--------|---------|

然后对每只标的输出完整分析卡片（Step 2-7），最后在 Step 9 给出统一的资金分配建议。

**1. 评分排名表：**

| 排名 | 代码 | 公司 | 总分 | 建议 | 信心度 |
|------|------|------|------|------|--------|
| 1 | XXXX.HK | 公司A | 82 | 强烈推荐申购 | 高 |
| 2 | YYYY.HK | 公司B | 65 | 可适量申购 | 中 |
| 3 | ZZZZ | 公司C | 45 | 建议观望 | 低 |

**建议等级（三档判定）：**
- ≥75: 强烈推荐申购（SUBSCRIBE）
- 50-74: 可适量申购 / 博首日收益（SUBSCRIBE FOR LISTING GAINS）
- <50: 不建议申购 / 建议观望（AVOID）

**2. 资金分配建议矩阵：**
- 可用资金总额（用户输入或默认 10 万港元/美元）
- 各标的建议分配比例
- 是否建议融资申购（港股）
- 现金 vs 融资方案对比（含利息成本）
- 预估回报率（乐观/中性/悲观）

**3. 打新策略建议：**
- 优先级排序与理由
- 时间窗口提醒（截止日期倒计时）
- 上市首日操作建议（持有/暗盘卖出/首日卖出/长期持有）
- 风险提示（破发风险、流动性风险、汇率风险、锁定期到期风险）

**保存路径：** `reports/ipo_subscription_YYYYMMDD.md`

## 中文输出

- 所有分析报告、摘要、建议使用**中文**输出
- 保留英文的情况：ticker 代码（00700.HK, AAPL）、专有名词首次出现时附英文（基石投资者 Cornerstone Investors）、数据源名称
